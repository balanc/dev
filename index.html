Love Letter to Git

Log Buttons

Dear Git

I surrender to you. My money, my material possessions, my time, all of it is yours. Every part of my being is here for you. If there is ever another lifetime beyond this, I would still choose you.

The way you lovingly care to watch over everything for me after I say git init fills me with an ever sunny disposition. When you start watching over my new directory, I know it’s in good hands because you’re always checking to make sure everything is just right for me. I ask about how things are with git status and you tell me what’s new. Thank you for always letting me know if I forgot to add a file to staging with git add <filename.extension>. You make it so easy on me when all I have to put is a dot to add everything new there. You’re too kind, really. I can’t tell you enough.

In the past, I avoided commitments. I truly did not want to hurt anyone, but in the end, I was hurting not only others but myself too. I want to change all of that. Please. I trust you so much, almost to a fault, but is that wrong? I don’t even care, as long as you stay by my side.

I’ve finally learned to commit and I will continue to love you as much as you love me. I looked at all of my early commits the other day with git log and I started thinking about it. It’s almost hilarious about how the timing is so coincidental that I finally commit to something in my life and I learn to commit all of the files on stage with git commit. I guess the first time I have ever officially committed was with that readme.md file and said git commit -m “first commit”. The experience itself was unusually satisfying. Perhaps that is when I first started to open up to you. Thank you for being patient with me.

Do you remember when I first forgot to put that "-m" option after “git commit”? It was extremely unnerving. I missed having you around. I didn’t know what to do without you there. You sent me to vim to announce my commit message, but it was so different from what I expected an editor to be. I don’t want to have to rely on you to do everything, so I’m committed (haha) to learn everything I can for you.so we can work together better. At the very least, I know that there are two modes that do things in the vim editor. Keys are commands in normal mode there, that’s the default mode when you first go in. You can apparently just hit the Esc key to be in normal mode too. The “I” key is insert mode. That’s when I can type my commit message. After I’m done, I can just press Esc to return to normal mode and use “:w” to save my message and then use “:q” to quit vim or I can just do both at the same time with “:wq” to save and quit and come back to you. As afraid as I was, I promise to learn how to take care of myself there. I mean, if I ever need or want to use an editor I’m familiar with, like nano, I’ll keep in mind what you said to me to change it, git config --global core.editor “nano”. I won’t back down, though. Like I said, I’m committed to learning.

The first time I knew I couldn’t live without you was when I accidentally deleted the file I was working on. Everything was gone. The panic that set in stirred to resentment and I almost gave up completely. Oh how you embrace me with a second wind, a miracle, second only to the gift of finding you. All of that pain will forever cease as long as I continue to use git checkout <commit hash> <filename> to get everything back.

The way that you let me branch off in a different direction, but still keep all of my original work is simply breathtaking. I can play around all I want with git branch <branchname> without fear of ruining anything. Sometimes it works out for the best and think I should git merge to my original work. I get lost with all of the changes I make sometimes. It’s unbelievably helpful how git diff can compare all of the changes that I’ve made between commits and even branches. I cannot thank you enough. Really, you spoil me rotten.
